## 二、🚨 Critical/高优先级问题

### 1. Logo 路径引用不一致（可能导致图片404）
**问题**: 子目录页面（`/cn/`、`/tw/`、`/admin/`）引用 `/brand/logo-dark-icon.png` 是绝对路径，但 styles.css 是 `/styles.css`（绝对路径）。
- **影响**: 如果部署时路径映射不一致，logo 可能无法显示
- **建议**: 统一使用绝对路径 `/brand/logo-dark-icon.png` 已正确，确认 Vercel 配置了静态文件服务

### 2. fade-in 动画可访问性问题
**问题**: 所有 `.fade-in` 元素初始 `opacity: 0`，如果用户：
- 禁用 CSS 动画（prefers-reduced-motion）
- 使用低端设备
- 浏览器渲染异常
- **内容将完全不可见**

**修复建议** (添加到 `styles.css`):
```css
@media (prefers-reduced-motion: reduce) {
  .fade-in {
    opacity: 1 !important;
    transform: none !important;
    animation: none !important;
  }
}
```

### 3. 缺少 OG/Twitter Card 图片
**问题**: `<meta property="og:image">` 和 Twitter Card 图片完全缺失。分享时不会有图片预览。

**建议添加**:
```html
<meta property="og:image" content="https://fidesorigin.com/brand/og-image.png">
<meta name="twitter:image" content="https://fidesorigin.com/brand/og-image.png">
<meta name="twitter:site" content="@fidesorigin">
```

### 4. 缺少结构化数据 (JSON-LD)
**问题**: 没有 Schema.org 标记，搜索引擎无法正确识别：
- Organization 信息
- Product/SoftwareApplication 信息
- ContactPoint

**建议添加** (在 `<head>` 中):
```html
<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "Organization",
  "name": "FidesOrigin",
  "url": "https://fidesorigin.com",
  "logo": "https://fidesorigin.com/brand/logo-dark-icon.png",
  "description": "Real-time on-chain compliance for tokenized assets",
  "sameAs": ["https://github.com/FintechGuy71/FidesOrigin"],
  "contactPoint": {
    "@type": "ContactPoint",
    "email": "contact@fidesorigin.com",
    "contactType": "sales"
  }
}
</script>
```

### 5. Trust Bar Emoji 旗帜显示不一致
**问题**: 🇭🇰 🇪🇺 🇺🇸 在不同平台/浏览器显示差异大：
- Windows 可能显示为黑白字母组合
- 部分Linux显示为国旗缩写

**建议**: 使用 SVG 国旗图标替代 emoji，或添加字体回退：
```css
.trust-item .flag {
  font-family: "Apple Color Emoji", "Segoe UI Emoji", "Noto Color Emoji", sans-serif;
}
```

## 三、🔶 Medium/中优先级优化

### 6. 未使用的 CSS 代码
**问题**: `styles.css` 中定义了大量未被使用的类：
- `.hero` (首页使用 `.brand-hero`)
- `.comparison`、`.comparison-card`、`.comparison-list`
- `.pillars`、`.pillar`
- `.features-grid`、`.feature-card`
- `.chains-grid`、`.chain-item`
- `.benchmark-grid`、`.benchmark-card`
- `.demo-box`、`.demo-input`、`.demo-result`
- `.risk-indicator`、`.risk-safe`、`.risk-low`、`.risk-high`

**影响**: 额外 10KB+ CSS 被下载但从未使用，浪费带宽。

**建议**: 清理未使用样式，或创建按需加载的模块化 CSS。

### 7. 缺少 `loading="lazy"` 图片优化
**问题**: Logo 图片没有 `loading="lazy"`：
```html
<img src="/brand/logo-dark-icon.png" alt="FidesOrigin">
```

**建议**: 对于首屏以下/非关键的图片添加 lazy loading：
```html
<img src="/brand/logo-dark-icon.png" alt="FidesOrigin" loading="lazy" decoding="async">
```

### 8. Footer "Docs — Coming Soon" 误导性设计
**问题**: 这是一个 `<span>` 不是 `<a>`，但视觉样式和周围链接相同，用户会误以为可点击。

**建议**: 改为灰色不可点击样式，或添加占位链接：
```html
<span style="display:block;color:var(--text-muted);font-size:0.85rem;padding:4px 0;cursor:default;opacity:0.6;">
  Docs — Coming Soon
</span>
```

### 9. 缺少 `<main>` 标签语义化
**问题**: 页面没有使用 `<main>` 标签包裹主要内容，影响屏幕阅读器导航。

**建议**: 将页面主体包裹在 `<main>` 中：
```html
<main id="main-content">
  <!-- all sections -->
</main>
```

### 10. Header 缺少 Skip Link（无障碍）
**问题**: 键盘用户无法快速跳转到主内容。

**建议**: 在 `<body>` 开头添加：
```html
<a href="#main-content" class="skip-link">Skip to main content</a>
```

### 11. 代码块缺少复制按钮
**问题**: "Built for AI Agents" 代码块是纯展示，没有复制功能。开发者需要手动选择复制。

**建议**: 添加一键复制按钮：
```javascript

```

## 四、🔹 Low/低优先级优化建议

### 12. 添加 Scroll-to-Top 按钮
**问题**: 页面较长，但没有快速返回顶部的按钮。

**建议**: 在右下角添加固定位置的返回顶部按钮，滚动时显示：
```html
<button class="scroll-top" onclick="window.scrollTo({top:0,behavior:'smooth'})">↑</button>
```

### 13. Stats 数字可考虑 Count-Up 动画
**问题**: 20,645+ 等数字是静态展示，没有动态效果。

**建议**: 数字进入视口时触发 count-up 动画，增强视觉冲击力：
```javascript

```

### 14. 添加 Newsletter 订阅入口
**问题**: 没有邮件订阅或社区加入入口，无法沉淀用户。

**建议**: 在 Footer 添加简单的邮件订阅表单（可对接 Resend/ConvertKit）。

**问题**: GitHub 链接是静态文本，没有显示实时 stars 数。

**建议**: 添加 GitHub Stars badge 或使用 API 获取实时数据。

### 16. 添加 Changelog 或 Blog 入口
**问题**: 产品持续迭代，但没有对外展示更新记录的地方。

**建议**: 在导航或 Footer 添加 "Changelog" 或 "Blog" 链接。

**问题**: `detectLang()` 使用 ipapi.co，如果用户通过 VPN 访问会误判。

**建议**: 优先使用 `navigator.language`，IP 作为后备：
```javascript

```

### 18. 缺少 `robots.txt` 和 `sitemap.xml`
**问题**: 没有搜索引擎爬虫指引文件。

**建议创建**:
- `robots.txt`: 允许所有爬虫，指向 sitemap
- `sitemap.xml`: 包含所有页面 URL（en, cn, tw, address-check, admin）

### 19. `<html>` 缺少 `dir` 属性
**问题**: 虽然当前语言不涉及 RTL，但作为最佳实践：
```html

```

### 20. 缺少 `theme-color` meta
**问题**: 移动浏览器地址栏颜色不匹配主题。
